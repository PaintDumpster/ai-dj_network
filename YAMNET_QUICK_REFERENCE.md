# YAMNet Classification Node - Quick Reference

## Overview

The `yamnet_classification` node uses ONNX Runtime to run Google's YAMNet audio classification model on waveforms generated by button presses.

## Architecture

```
Arduino (10 buttons) → reader → arduino_data topic
                                       ↓
                              build_waveform → waveform CSV → /tmp/
                                                                  ↓
                                                        yamnet_classification
                                                                  ↓
                                                    classification_results topic
```

## Current Implementation Status

✅ **Completed:**
- ONNX Runtime integration framework
- Waveform loading from CSV files
- Audio resampling (44.1kHz → 16kHz)
- Preprocessing pipeline (padding, normalization)
- Inference engine
- Top-5 results publishing
- Configurable parameters

⚠️ **Requires Setup:**
- Install ONNX Runtime library (see [YAMNET_SETUP_GUIDE.md](YAMNET_SETUP_GUIDE.md))
- Download/convert YAMNet model to ONNX format
- Place model files in `models/` directory

🔧 **Future Enhancements:**

- ROS service interface for on-demand classification
- Direct topic subscription (instead of file-based)
- Real-time streaming classification
- Multi-waveform batch processing

## Installation

### 1. Install ONNX Runtime

```bash
# Quick install (Ubuntu/Debian)
cd ~/Downloads
wget https://github.com/microsoft/onnxruntime/releases/download/v1.24.2/onnxruntime-linux-x64-1.24.2.tgz
tar -xzf onnxruntime-linux-x64-1.24.2.tgz
sudo mkdir -p /opt/onnxruntime
sudo cp -r onnxruntime-linux-x64-1.24.2/* /opt/onnxruntime/
sudo ln -sf /opt/onnxruntime/lib/libonnxruntime.so* /usr/local/lib/
sudo ldconfig
```

### 2. Get YAMNet Model

```bash
cd ~/iaac/ai4all/rosnetwork/models
pip3 install tensorflow tf2onnx tensorflow-hub

# Create and run conversion script (see YAMNET_SETUP_GUIDE.md)
```

### 3. Build

```bash
cd ~/iaac/ai4all/rosnetwork
colcon build --packages-select cpp_pkg
source install/setup.bash
```

## Input/Output Specifications

### Input
- **Format**: CSV file with columns: `time,amplitude`
- **Sample Rate**: Any (automatically resampled to 16kHz)
- **Duration**: Any (padded/truncated to 0.975s = 15,600 samples at 16kHz)
- **Amplitude Range**: Any (automatically normalized to [-1, 1])

### Output (topic: `/classification_results`)
- **Message Type**: `std_msgs/msg/String`
- **Content**: Top-5 classification results with scores
- **Format**:
  ```
  Classification Results:
  1. <class_name>: <score>
  2. <class_name>: <score>
  ...
  ```

## YAMNet Classes

YAMNet recognizes 521 AudioSet classes:

**Music**: Various instruments, genres, musical concepts
**Speech**: Conversation, narration, singing
**Animals**: Dogs, cats, birds, insects
**Natural Sounds**: Rain, wind, water, thunder
**Human Sounds**: Laughter, crying, breathing, footsteps
**Mechanical**: Engines, alarms, tools, vehicles
**And 500+ more...**

Full list in: `models/yamnet_class_names.txt`

## Technical Details

### ONNX Runtime
- **Version**: 1.24.2 (latest stable)
- **API**: C++ API (onnxruntime_cxx_api.h)
- **Execution**: CPU (default), can be configured for GPU

### YAMNet Model
- **Architecture**: MobileNetV1-based audio event classifier
- **Training Data**: AudioSet (2 million YouTube clips)
- **Parameters**: ~3.7M
- **Model Size**: ~3.5 MB (ONNX format)

### Audio Processing Pipeline
1. Load waveform from CSV
2. Resample to 16kHz (linear interpolation)
3. Pad/truncate to 15,600 samples (0.975s)
4. Normalize amplitude to [-1, 1]
5. Create ONNX tensor
6. Run inference
7. Sort predictions by score
8. Return top-K results

## Troubleshooting

### Build Error: ONNX Runtime not found
```bash
# Check installation
ls /opt/onnxruntime/lib/libonnxruntime.so
ldconfig -p | grep onnxruntime

# If missing, reinstall following YAMNET_SETUP_GUIDE.md
```

### Runtime Error: Model file not found
```bash
# Verify model exists
ls -lh ~/iaac/ai4all/rosnetwork/models/yamnet.onnx

# If missing, run conversion script
cd ~/iaac/ai4all/rosnetwork/models
python3 convert_yamnet.py
```

### Low Classification Scores
- Ensure audio is clear and not clipped
- Check that button sounds are distinctive
- Verify waveform CSV format is correct
- Try different sound files in `sounds/` folder

## Performance Notes

- **Inference Time**: ~50-100ms per waveform (CPU, depends on hardware)
- **Memory Usage**: ~50MB (model loaded in RAM)
- **Throughput**: Can process ~10-20 waveforms/second
